Memory Tracker Fix - Cleanup Sequence Resolution

Date: 26th August 2025

Author: Ivy Mycelia

Great news! I've successfully fixed the memory pointer issue that was causing Myco to print out warning messages when running *.myco files.

Root Cause: The cleanup sequence in main.c was calling memory_tracker_cleanup() before cleanup_implicit_functions(), which cleared all tracked allocations before we could properly free the operator_map.

The Fix: I reordered the cleanup sequence so that:

  1. First: cleanup_implicit_functions() - frees tracked operator_map memory
  2. Second: cleanup_loop_execution_state() - frees tracked loop state memory
  3. Third: parser_free_ast() - frees tracked AST memory
  4. Last: memory_tracker_cleanup() - clears the memory tracker

This ensures that all tracked memory is properly freed before the memory tracker itself is cleaned up, eliminating the warning messages and preventing potential memory leaks.

The fix maintains the integrity of the memory tracking system while ensuring proper cleanup order. Myco should now run *.myco files without any memory-related warnings!
