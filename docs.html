Myco Language Documentation

Language Overview

Myco is a whimsical, mystical programming language rooted deep in the forest floor. Think C meets Lua — but with mushrooms. It's a lightweight, expressive scripting language designed for simplicity, readability, and just a touch of magic.

Current Version: v1.4.0 - Float Support & Library Import System

Key Design Principles

  • Simplicity: Minimal syntax with clear, readable code
  • Performance: Efficient execution with minimal overhead
  • Portability: Cross-platform compatibility
  • Embeddability: Easy integration into existing applications
  • Magic: Whimsical and mystical design philosophy

Latest Features

  • Float Support: Full floating-point arithmetic and mixed operations
  • Library Import System: Namespace protection and modular code
  • True/False Keywords: Boolean literals with backward compatibility
  • Enhanced Math Library: 15+ mathematical functions
  • Utility Library: 20+ utility functions for debugging
  • Arrow Syntax: Modern function return type annotations
  • Switch/Case Statements: Expression-based case matching
  • Try-Catch Error Handling: Graceful error management

Getting Started

Installation

Myco is currently in development. To build from source:

Build Commands 
git clone https://github.com/IvyMycelia/Myco.git
cd myco/myco
make

Running Programs

Execution 
./myco filename.myco

Create a file named hello.myco:

hello.myco

    print("Welcome to the woods,", person, "!");

    return pow(base, exponent);
end

greet(name);

Advanced Features

Library Import System

Myco v1.4.0 introduces a powerful library import system with namespace protection:

Library Imports 
# Import math library with namespace
use math as m;

# Import utility library
use util as u;

# Use math functions with namespace
let pi = m.PI;                  # 3.14159
let e = m.E;                    # 2.71828
let abs_val = m.abs(-5);        # 5
let power = m.pow(2, 8);        # 256

# Use utility functions
let user = {"name": "Alice", "age": 25};
u.debug(user);                  # Detailed object inspection
let user_type = u.type(user);   # Returns "Object"
let has_name = u.has(user, "name"); # Returns 1 (true)

Lambda Functions

Myco supports lambda functions for functional programming:

Lambda Functions 
let numbers = [1, 2, 3, 4, 5];

# Filter even numbers using lambda
let isEven = x => x % 2 == 0;
let evens = filter(numbers, isEven);

# Transform numbers using lambda
let double = x => x * 2;
let doubled = map(numbers, double);

# Reduce array to sum
let add = (acc, x) => acc + x;
let sum = reduce(numbers, 0, add);

Complete Examples

Simple Calculator

Calculator 
func calculator():
    print("Simple Calculator");
    print("1. Add");
    print("2. Subtract");
    print("3. Multiply");
    print("4. Divide");
    
    let choice = input_int("Enter choice (1-4): ");
    let a = input_float("Enter first number: ");
    let b = input_float("Enter second number: ");
    
    switch choice:
        case 1:
            print("Result: " + (a + b));
        case 2:
            print("Result: " + (a - b));
        case 3:
            print("Result: " + (a * b));
        case 4:
            try:
                print("Result: " + (a / b));
            catch error:
                print("Error: Division by zero");
            end
        default:
            print("Invalid choice");
    end
end

calculator();

Data Processing

Data Processing 
# Process user data
let users = [
    {"name": "Alice", "age": 25, "score": 95},
    {"name": "Bob", "age": 30, "score": 87},
    {"name": "Charlie", "age": 28, "score": 92}
];

# Filter high-scoring users
let high_scorers = filter(users, user => user["score"] >= 90);

# Calculate average age
let total_age = reduce(users, 0, (acc, user) => acc + user["age"]);
let avg_age = total_age / len(users);

print("High scorers:");
for user in high_scorers:
    print("- " + user["name"] + " (Score: " + user["score"] + ")");
end

print("Average age: " + avg_age);

Best Practices

Code Organization

  • Use meaningful variable names that describe their purpose
  • Group related functions together in logical sections
  • Add comments to explain complex logic
  • Use consistent indentation for readability

Error Handling

  • Always use try-catch for operations that might fail
  • Provide meaningful error messages to help debugging
  • Handle edge cases like division by zero

Performance

  • Use appropriate data structures for your use case
  • Avoid unnecessary loops when built-in functions exist
  • Use library functions instead of reimplementing common operations

Troubleshooting

Common Issues

Common Problems 
# Missing semicolon
let x = 5  # Error: missing semicolon
let x = 5; # Correct

# Undefined variable
print(undefined_var);  # Error: variable not defined

# Type mismatch
let result = "5" + 3;  # Result: "53" (string concatenation)
let result = to_int("5") + 3;  # Result: 8 (numeric addition)

Debugging Tips

  • Use the debug() function to inspect variables and objects
  • Add print statements to trace program flow
  • Check variable types with the type() function
  • Use try-catch to identify where errors occur
