Myco Language Documentation

Language Overview

Myco is a modern, lightweight programming language designed for simplicity and expressiveness. It features dynamic typing, object-oriented capabilities, functional programming with lambda functions, and a clean syntax inspired by Lua and Python.

Version Information

Current Version: 1.6.0 - Language Maturity & Developer Experience

License: MIT

Repository: https://github.com/IvyMycelia/myco

Latest Features

Comprehensive float support, enhanced math functions, clear type system, comprehensive utility library, arrow syntax for functions, switch/case statements, try/catch error handling, system integration libraries, language maturity features

Key Design Principles

  • Simplicity: Minimal syntax with clear, readable code
  • Performance: Efficient execution with minimal overhead
  • Portability: Cross-platform compatibility
  • Embeddability: Easy integration into existing applications
  • Modern Features: Lambda functions, type system, comprehensive libraries

Getting Started

Prerequisites

  • GCC compiler
  • Make build system
  • Git (for source installation)

Building from Source

Build Commands 
git clone https://github.com/TrendyBananaYT/myco.git
cd myco/myco
make

Running Programs

Execution 
./myco filename.myco

Your First Myco Script

Create a file named hello.myco:

hello.myco 
print("Hello, Myco World!");

let name = "Developer";
print("Welcome, " + name + "!");

Run it with: ./myco hello.myco

Type System

Myco features a clear, descriptive type system that returns meaningful names instead of cryptic codes:

Type Checking Examples 
# Type checking with clear names
let num_type = type(42);        # Returns: "Integer"
let str_type = type("hello");   # Returns: "String"
let arr_type = type([1,2,3]);  # Returns: "Array"
let obj_type = type({x:1});    # Returns: "Object"

# Boolean type checks
let is_number = is_num(42);     # Returns: 1 (True)
let is_string = is_str("test"); # Returns: 1 (True)
let is_array = is_arr([1,2]);   # Returns: 1 (True)
let is_object = is_obj({x:1}); # Returns: 1 (True)

Functions

Myco supports two syntaxes for function declarations:

Traditional Colon Syntax

Traditional Function Syntax
func functionName(parameter1, parameter2):
    # function body
    return value;
end

New Arrow Syntax

Arrow Function Syntax
func functionName(parameter1, parameter2) -> returnType:
    # function body
    return value;
end

Lambda Functions

Myco now supports lambda functions (arrow functions) for functional programming:

Lambda Examples
# Single parameter
let double = x => x * 2;
let result = double(5);  # 10

# Multiple parameters
let add = (a, b) => a + b;
let sum = add(10, 20);  # 30

# No parameters
let getFive = () => 5;
let value = getFive();  # 5

Implicit Functions

Myco now supports true implicit functions in the Python style:

Implicit Function Examples
# No parameter type annotations needed
func add_numbers(a, b):           # Types are inferred automatically
    return a + b;
end

# Functions can return nothing implicitly
func process_data(x):
    print("Processing:", x);
    # No return statement = implicit return of 0
end

File Processing

file_processor.myco 
func process_file(filename):
    let file = file_open(filename, "r");
    
    if file == 0:
        print("Error: Could not open file " + filename);
        return;
    end
    
    let line_count = 0;
    let word_count = 0;
    let char_count = 0;
    
    while 1:
        let line = file_read_line(file);
        
        if line == "":
            break;
        end
        
        line_count = line_count + 1;
        char_count = char_count + len(line);
        
        let words = split(line, " ");
        word_count = word_count + len(words);
    end
    
    file_close(file);
    
    print("File Statistics:");
    print("Lines: " + line_count);
    print("Words: " + word_count);
    print("Characters: " + char_count);
end

process_file("input.txt");

Best Practices

Code Style

  • Indentation: Use consistent indentation (2 or 4 spaces)
  • Naming: Use descriptive names for variables and functions
  • Comments: Add comments for complex logic
  • Functions: Keep functions small and focused

Performance Tips

  • Loops: Avoid nested loops when possible
  • Memory: Close files and free resources
  • Strings: Use string concatenation sparingly in loops
  • Data structures: Choose appropriate data structures for your use case

Error Handling Example 
func safe_divide(a, b):
    if b == 0:
        print("Error: Division by zero");
        return 0;
    end
    
    return a / b;
end

# Using try-catch for better error handling
try:

Troubleshooting

Common Issues

Common Problems 
# Issue: "Variable not defined"
# Solution: Always declare variables with 'let'
let x = 5;  # Correct
y = 10;     # Wrong - will cause error

# Issue: "Function not found"
# Solution: Check function name spelling and ensure it's defined
func my_function():
    print("Hello");
end

my_function();  # Correct
myFunction();  # Wrong - different name

# Issue: "Index out of bounds"
# Solution: Check array bounds before accessing
let list = [1, 2, 3];
if len(list) > 0:
    print(list[0]);  # Safe access
end

Debugging Tips

  • Use print statements: Add print statements to track variable values
  • Check syntax: Ensure all blocks are properly closed with 'end'
  • Verify types: Make sure you're using the right data types
  • Test incrementally: Test small parts of your code before building larger programs

Troubleshooting

Common Issues and Solutions

Performance Issues

Problem: Myco programs run slowly

Solutions:

  • Use --debug flag to identify bottlenecks
  • Check for unnecessary loops or calculations
  • Use built-in functions instead of custom implementations
  • Profile with debug.start_timer() and debug.end_timer()

Syntax and Parsing Issues

Problem: "Expected semicolon" errors

Solution: Ensure all statements end with semicolons

Correct Syntax 
let x = 5;        # Correct
print("Hello");   # Correct
let y = 10        # Missing semicolon - ERROR

Runtime Issues

Problem: "Undefined variable" errors

Solutions:

  • Check variable spelling and case
  • Ensure variables are declared before use
  • Use debug() to inspect variable environment

Debugging Techniques

Debugging Example 
# Start performance profiling
debug.start_timer();

# Your code here
for i in 1..1000000:
    let result = i * 2;

# End profiling and get results
let elapsed = debug.end_timer();
print("Execution time:", elapsed, "ms");

# Get memory statistics
let stats = debug.get_stats();

Best Practices for Error Prevention

  1. Always validate input: Check user input and function parameters
  2. Use try-catch blocks: Wrap risky operations in error handling
  3. Check array bounds: Validate indices before array access
  4. Handle edge cases: Consider what happens with empty arrays, null values, etc.
  5. Test thoroughly: Use the comprehensive unit test suite as a reference
